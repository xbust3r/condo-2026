# Observations — lectura humana de `condo-py`

Esta carpeta existe para traducir la arquitectura del proyecto a lenguaje humano.

Aquí no se intenta impresionar con palabras grandes.
Se intenta explicar:

- qué se observó,
- por qué importa,
- qué está bien,
- qué sigue en deuda,
- y cómo pensar cambios futuros sin ensuciar el diseño.

## Orden recomendado de lectura

1. `architecture-observations.md`
2. `domain-vs-application.md`
3. `recommendations-explained.md`
4. `junior-guide.md`

## Alcance

Estos archivos describen **`condo-py`**, no la teoría genérica.
La teoría base vive en `../new-standard/`.

`new-standard/` responde: “cómo debería pensarse una arquitectura DDD reusable”.

`observations/` responde: “qué significa eso aplicado al proyecto que realmente tienes enfrente”.

## Idea central

Un developer que entienda estas observaciones deja de copiar carpetas por reflejo y empieza a mover piezas con criterio.
